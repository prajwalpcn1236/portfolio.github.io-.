üóÉÔ∏è 1. Hash Maps

‚úÖ Why Used:

For fast key-value lookups in inventory management.

Enables O(1) average-time access to product information by SKU or ID.

‚è±Ô∏è Time Complexity:

Insert: O(1) average

Search: O(1) average

Delete: O(1) average

üß† Space Complexity:

O(n) where n is the number of key-value pairs

Load factor affects performance and space usage

Implementation Code 


#include <iostream>
#include <unordered_map>
using namespace std;

int main() {
    unordered_map<string, int> fruitCount;

    // Inserting values
    fruitCount["apple"] = 3;
    fruitCount["banana"] = 5;
    fruitCount["orange"] = 2;

    // Accessing elements
    cout << "Apple count: " << fruitCount["apple"] << endl;

    // Iterating over the map
    for (auto& pair : fruitCount) {
        cout << pair.first << ": " << pair.second << endl;
    }

    return 0;
}

üå≥ 2. Red-Black Trees

‚úÖ Why Used:

For maintaining sorted order while ensuring balanced tree operations.

Used in warehouse management for sorted inventory access and range queries.

‚è±Ô∏è Time Complexity:

Insert: O(log n)

Search: O(log n)

Delete: O(log n)

üß† Space Complexity:

O(n) where n is the number of nodes

Self-balancing property ensures optimal height

Implementation Code 


#include <iostream>
#include <set>
using namespace std;

int main() {
    set<int> rbTree;

    // Insert elements
    rbTree.insert(10);
    rbTree.insert(20);
    rbTree.insert(15);
    rbTree.insert(5);

    // Iterating elements (in sorted order)
    cout << "Elements in Red-Black Tree:" << endl;
    for (int val : rbTree) {
        cout << val << " ";
    }
    cout << endl;

    // Check if element exists
    if (rbTree.find(15) != rbTree.end()) {
        cout << "15 found in the tree." << endl;
    }

    return 0;
}

üóÑÔ∏è 3. B+ Trees

‚úÖ Why Used:

For efficient database storage and retrieval operations.

Optimized for systems that read and write large blocks of data, perfect for warehouse databases.

‚è±Ô∏è Time Complexity:

Insert: O(log n)

Search: O(log n)

Range queries: O(log n + k) where k = result size

üß† Space Complexity:

O(n) where n is the number of records

High fanout reduces tree height and improves cache performance

Implementation Code 

// Minimal B+ Tree node example in C++

#include <iostream>
#include <vector>

using namespace std;

const int ORDER = 3; // B+ Tree order (max children per node)

struct BPlusNode {
    bool isLeaf;
    vector<int> keys;
    vector<BPlusNode*> children;
    BPlusNode* next; // Pointer to next leaf (for leaf nodes)

    BPlusNode(bool leaf) : isLeaf(leaf), next(nullptr) {}
};

int main() {
    // Just create a leaf node for demonstration
    BPlusNode* leaf = new BPlusNode(true);

    // Insert some keys (simplified, no balancing here)
    leaf->keys.push_back(10);
    leaf->keys.push_back(20);
    leaf->keys.push_back(30);

    cout << "Leaf node keys: ";
    for (int k : leaf->keys) {
        cout << k << " ";
    }
    cout << endl;

    return 0;
}
