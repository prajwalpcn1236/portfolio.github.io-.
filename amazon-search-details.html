üìò 2. TF-IDF (Term Frequency - Inverse Document Frequency)

‚úÖ Why Used:

To weigh important terms more than common ones.

Helps in ranking product descriptions based on relevance to query.

‚è±Ô∏è Time Complexity:

TF computation: O(N * T) where N = number of documents (products), T = average number of terms per doc

IDF computation: O(N) (once per term)

Ranking query: O(Q) where Q = number of query terms

üß† Space Complexity:

TF matrix: O(N * T)

IDF table: O(V) where V = vocabulary size

üìò 3. Inverted Index

‚úÖ Why Used:

To instantly locate all documents (products) containing a given term.

Efficiently supports full-text search at scale.

‚è±Ô∏è Time Complexity:

Index construction: O(N * T) where N = docs, T = tokens per doc

Query retrieval: O(1) (with hashmap) per term

AND queries (e.g., "wireless headphones"): linear in the size of the posting lists

üß† Space Complexity:

O(V + P)

V = vocabulary

P = total postings = number of term-doc pairs
